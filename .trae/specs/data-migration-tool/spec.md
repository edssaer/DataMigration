# 数据库表迁移工具 - 产品需求文档

## Overview
- **Summary**: 一个可视化的数据库表迁移工具，支持从源数据库查询数据并导入到目标数据库生成新表，可自定义表结构和属性。
- **Purpose**: 简化数据库间的数据迁移流程，提供直观的界面操作，减少手动SQL编写的复杂度和错误率。
- **Target Users**: 数据库管理员、开发人员、数据分析师等需要进行数据库间数据迁移的技术人员。

## Goals
- 支持多种数据库类型的连接和迁移
- 提供可视化界面配置数据源和目标数据库
- 支持自定义SQL查询获取源数据
- 支持配置目标表的结构和属性
- 提供迁移过程的实时状态和日志
- 支持迁移任务的保存和复用

## Non-Goals (Out of Scope)
- 不支持实时同步功能
- 不支持跨网络环境的复杂部署
- 不支持大数据量(>1000万条)的高性能迁移
- 不支持非关系型数据库

## Background & Context
- 传统数据库迁移需要手动编写SQL脚本，操作复杂且容易出错
- 现有工具往往缺乏直观的可视化界面，对非专业人员不友好
- 不同数据库间的语法差异增加了迁移难度

## Functional Requirements
- **FR-1**: 数据源管理
  - 支持配置多个源数据库连接
  - 支持常见数据库类型(MySQL, PostgreSQL, SQL Server等)
  - 支持测试连接有效性
- **FR-2**: 数据查询配置
  - 提供SQL编辑器编写查询语句
  - 支持查询结果预览
  - 支持保存常用查询语句
- **FR-3**: 目标表配置
  - 支持自定义目标表名称
  - 支持配置表字段的类型、长度、约束等属性
  - 支持设置主键和索引
- **FR-4**: 迁移执行
  - 执行数据迁移并显示进度
  - 记录迁移日志和错误信息
  - 支持暂停和取消迁移任务
- **FR-5**: 任务管理
  - 支持保存迁移任务配置
  - 支持查看历史迁移记录
  - 支持导出和导入任务配置

## Non-Functional Requirements
- **NFR-1**: 性能
  - 迁移10万条数据的时间不超过30秒
  - 界面响应时间不超过1秒
- **NFR-2**: 可用性
  - 提供友好的错误提示和操作引导
  - 支持断点续传
- **NFR-3**: 安全性
  - 数据库连接信息加密存储
  - 支持HTTPS协议

## Constraints
- **Technical**: 后端使用Go或Python，前端使用Web技术
- **Business**: 本地部署，使用root用户操作数据库
- **Dependencies**: 需要数据库驱动支持多种数据库类型

## Assumptions
- 本地数据库环境已配置好，可使用root用户和指定密码访问
- 目标数据库NJYY已存在
- 网络环境稳定，无需考虑网络中断情况

## Acceptance Criteria

### AC-1: 数据源配置
- **Given**: 用户打开工具界面
- **When**: 输入数据库连接信息并点击测试连接
- **Then**: 显示连接成功/失败状态
- **Verification**: `programmatic`

### AC-2: 数据查询
- **Given**: 已配置源数据库连接
- **When**: 编写SQL查询并执行
- **Then**: 显示查询结果预览
- **Verification**: `programmatic`

### AC-3: 目标表配置
- **Given**: 已获取查询结果
- **When**: 配置目标表结构和属性
- **Then**: 显示配置预览
- **Verification**: `human-judgment`

### AC-4: 迁移执行
- **Given**: 已完成所有配置
- **When**: 点击开始迁移
- **Then**: 显示迁移进度和结果
- **Verification**: `programmatic`

### AC-5: 任务管理
- **Given**: 完成迁移任务
- **When**: 保存任务配置
- **Then**: 任务配置被成功保存
- **Verification**: `programmatic`
